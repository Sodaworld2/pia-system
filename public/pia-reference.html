1. What Is PIA?

PIA is your personal AI operations platform. It runs on the computers in your studio and manages a team of AI agents on your behalf — each agent has a name, a personality, a defined job, and a schedule. You direct strategy. The agents handle execution.

Think of it as hiring a staff of specialists who never sleep, never forget, and never need managing — except when you want to give them new direction.

The One-Line Version You tell Fisher2050 what matters this week. Fisher assigns tasks to the right agents. Agents do the work. Tim Buc files everything. Eliyahu reads it all and briefs you every morning at 6am.

PIA is not a chatbot. It is infrastructure — like a server rack that runs your business logic. The difference is the logic is intelligent and the workers are AI agents.

2. The Three Machines

Machine Nickname Role What It Runs Status
M1 Izzit7 Hub — the brain Dashboard, database, Fisher2050, Eliyahu, Tim Buc, all cron jobs, PIA control centre Online
M2 soda-monster-hunter Worker — execution Runs agents dispatched by M1 (Ziggi, Bird Fountain). Connects back to M1 as its worker. Online — synced Feb 22
M3 soda-yeti Worker — execution Runs agents dispatched by M1 (Farcake, Andy, Wingspan). Reachable via Tailscale. Reachable — SSH key pending

All three machines talk to each other over Tailscale — a private encrypted network that connects them wherever they are, as if they're on the same desk. M1 is the controller. M2 and M3 receive instructions from M1 and report results back.

Important M1 must always be running first. M2 and M3 connect to M1 — if M1 is off, the workers have no hub to connect to.

4. How a Day Works

When fully running, this is what happens in a 24-hour cycle with no input from you:

TimeWhoWhat Happens
6:00amEliyahuReads Tim Buc's records from yesterday. Writes your morning briefing. Sends it to your email before you wake up.
9:00amFisher2050Reviews the task list. Assigns work to available agents. Schedules runs for the day. Posts a standup summary.
During dayFarcake, Andy, Bird Fountain, Wingspan, Coder MachineConsume jobs from the queue. Research, write, build, create. Report results back to M1.
On completionTim BucEvery time any agent finishes, Tim Buc wakes and files the session log to the Records database.
2:00amZiggiReviews everything completed during the day. Scores quality. Flags anything that needs revision.
6:00amEliyahuReads Ziggi's reviews + agent records. Writes the next morning briefing. Cycle continues.

You step in when: you want to redirect strategy, review something Ziggi flagged, or approve a piece of work before it ships. The rest runs without you.

Mic (strategy) → Fisher2050 (plan) → Farcake / Andy / Bird Fountain (execute) → Ziggi (review) → GumballCMS (publish) → Tim Buc (archive) → Eliyahu (brief)

7. Cost Model

PIA is designed to use the cheapest AI model capable of doing the job. This is the waterfall:

TierModelCostUsed For
Free Ollama (local) $0.00 Routing decisions, simple summaries, internal agent-to-agent messages. Runs on your own hardware.
Cheap Claude Haiku ~$0.001 per task Tim Buc archiving, Monitor alerts, short research tasks, anything that doesn't need deep reasoning.
Standard Claude Sonnet ~$0.01–0.05 per task Fisher2050 planning, Andy writing, Farcake research, Ziggi quality reviews.
Premium Claude Opus ~$0.10–0.50 per task Complex architectural decisions, Eliyahu's deep synthesis, Coder Machine on hard problems only.

The AI router (src/ai/ai-router.ts) is already built and decides which tier to use based on task complexity. Itai will monitor actual spend and alert when it exceeds budgets.

8. Daily Operations — Quick Reference

Checking Everything is Running

Open your browser on M1: http://localhost:3000/mission-control.html

You should see: 2–3 machines listed (green dot = online), all services showing active.

Restarting PIA on M1

Open a terminal on M1 and run:

pm2 restart ecosystem.config.cjs --update-env

Restarting M2 (remotely from M1)

Open a terminal on M1 and run:

ssh user@100.127.165.12 "cd C:/Users/User/Documents/GitHub/pia-system && npx pm2 restart ecosystem.config.cjs --update-env"

Running Fisher Manually

In the dashboard (localhost:3000/mission-control.html), find the Fisher2050 Scheduler panel (bottom right). Click Standup or Summary.

Viewing Agent Session Logs

Dashboard → click any agent → Session Log panel shows real-time output.

Viewing All Filed Records (Tim Buc's archive)

Dashboard → Tim Buc Records panel (bottom right) → click Refresh.

9. Troubleshooting

ProblemMost Likely CauseFix
Agent spawns but does nothing / exits immediately No API credits in Anthropic account Add credits at console.anthropic.com/settings/billing
M2 or M3 not showing in dashboard Worker is offline or not connected to Tailscale Check Tailscale on that machine. Restart PIA: npx pm2 restart ecosystem.config.cjs --update-env
Eliyahu emails not arriving SendGrid key missing from .env Add SENDGRID_API_KEY to M1's .env file
Fisher crons not firing at scheduled time API credits or agent spawn error Check PM2 logs: pm2 logs pia-hub --lines 50
Dashboard shows "Disconnected" PIA server is not running on M1 Run on M1: pm2 restart ecosystem.config.cjs --update-env
Agent seems to ignore its personality / acts generic Soul wiring gap — souls are stored but not injected into spawns yet This is the next code fix planned. See SOUL_ENRICHMENT_SPEC.md.
M1 running slow Too many PM2 processes or memory pressure Run pm2 status to see all processes. Restart if uptime is very long.
