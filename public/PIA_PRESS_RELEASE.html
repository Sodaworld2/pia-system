SodaLabs Launches PIA: The World's First
Commercially Deployed AI Workforce Platform
with Persistent Agent Identity and Memory

Platform gives businesses a team of 12 named AI agents — each with a defined role, persistent memory, and individual email address — running on privately-owned hardware at a fraction of traditional staffing costs. First deployments operational in Johannesburg, South Africa.

Johannesburg, South Africa, 22 February 2026 — SodaLabs, a South African technology studio specialising in AI systems architecture, today announced the commercial launch of PIA (Project Intelligence Agent), a multi-machine AI orchestration platform that gives businesses access to a team of twelve specialised AI agents — each with a persistent identity, growing memory, and individual email address — deployed on the client's own hardware and running continuously without human supervision.

Unlike conventional AI assistant products that begin each session with no memory of prior interactions, PIA agents accumulate institutional knowledge across every session they run. The platform's proprietary Soul System assigns each agent a defined personality, professional role, relationship structure, operational rules, and memory layer that persists indefinitely — enabling agents to function as consistent, evolving team members rather than generic tools that must be re-trained with each use.

"The problem with every AI tool on the market today is that it forgets you the moment you close the window. We built PIA to solve that. Every agent on your team knows your business, knows your standards, knows your clients. They get better the longer they work with you — exactly the way a trusted employee does." — Mic Louw, Founder, SodaLabs

How It Works

PIA operates across a fleet of three to five computers connected via encrypted mesh networking. The hub machine — designated M1 — runs the platform's orchestration layer, dashboard, and strategic agents including Fisher2050 (Project Manager), Eliyahu (Intelligence Analyst), and Tim Buc (Records Manager). Worker machines run specialist agents on a scheduled or event-triggered basis: Farcake (Research), Andy (Editorial), Ziggi (Quality Review), Wingspan (Presentations), and Bird Fountain (Design Production).

At the core of the platform is PIA's Intelligence Pipeline — a self-reinforcing improvement loop in which every completed agent session is automatically archived and analysed: the archivist agent (Tim Buc) files all session logs, the analyst agent (Eliyahu) reads the organised records and synthesises patterns, and these patterns are fed back into the project manager's (Fisher2050's) task briefings. The result is a system that improves its understanding of the client's business with each passing week, without any manual intervention.

All data remains on the client's hardware. PIA does not route client information through third-party cloud servers. This architecture addresses a critical concern for professional services firms, legal practices, and any organisation handling confidential client information.

Key Differentiators from Existing AI Platforms
  • Persistent Agent Identity: Proprietary Soul System gives each agent a permanent personality, memory, goals, and relationship structure across all sessions — not achievable on any current commercial platform including OpenAI Assistants, Microsoft Copilot Studio, or CrewAI.
  • Private Hardware Deployment: The entire platform runs on client-owned hardware. No data leaves the client's network. No monthly per-seat cloud subscription. One deployment, permanently client-owned.
  • Calendar-Triggered Scheduling: Fisher2050 manages agent deployment through a calendar system — scheduling agent spawns based on project requirements, machine availability, and task dependencies. Eliminates manual agent triggering.
  • Intelligence Loop: Tim Buc → Records DB → Eliyahu → Fisher2050 creates a self-improving feedback cycle. Agent quality and contextual understanding improve with every session completed, without human configuration.
  • Individual Agent Email Addresses: Each agent has a real, monitored email address (e.g., fisher2050@sodalabs.ai). Clients can email a task directly to an agent and receive output without opening any dashboard — a first in commercial AI systems.
  • Multi-Machine Fleet Orchestration: PIA coordinates agents across multiple physical machines connected via WireGuard-encrypted mesh network, enabling parallel execution at scale not achievable on single-machine frameworks.
Availability and Engagement Model

PIA is available as a custom deployment engagement rather than a SaaS subscription. Each deployment is configured specifically for the client's business, with agents trained on the client's voice, standards, and operational context. First commercial deployments are operational in Johannesburg. SodaLabs is accepting engagement inquiries from professional services firms, creative agencies, and technology businesses internationally.

The company has not disclosed revenue figures for initial commercial deployments. SodaLabs is a privately held studio. No venture capital funding has been announced.
